## 已有 Agent

### 标书助手

> 路径：[标书/](标书/)

从招标文件到投标文件的全流程自动化助手，支持政务服务类、工程实施类等多种项目类型。

**核心能力：**

- 自动解析并分类招标文件（PDF / Word / 图片 / 文本）
- 按项目类型路由到专业分析 skill
- 筛选、评估、推荐合作厂商
- 生成完整可用的标书初稿
- 沉淀用户偏好与历史 brief，持续提升命中率

**执行流程：**

```
招标文件输入
    │
    ▼
① 路由分析          识别项目类型（政务/工程/其他/厂商筛选）
    │
    ▼
② 专业分析          提取评分点、拆解结构、输出优化建议
    │
    ▼
③ 厂商筛选          推荐分包商/合作厂商，含资质、报价、风险
    │
    ▼
④ 标书生成          结合历史 brief 与用户偏好，输出完整初稿
```

## 设计约定

新增 Agent 时遵循以下结构：

```
<agent-name>/
├── CLAUDE.md          # 总控：角色定义、工作流、场景判断规则
├── README.md          # 人类可读使用说明
├── agents/            # 流程代理（串联多个 skill 的编排层）
├── skills/            # 单一职责 skill（可独立调用）
└── <记录目录>/        # 运行时沉淀：brief、偏好、历史产物
```

**原则：**

- `CLAUDE.md` 是 Agent 的入口和约束层，不直接执行具体分析
- `skills/` 中每个文件只做一件事，可被多个 Agent 复用
- `agents/` 负责编排流程，读取 `CLAUDE.md` 后串联各 skill
- 运行产物（brief、偏好记录）存入专属目录，支持跨次调用
