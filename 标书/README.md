# agents 使用说明

本目录用于存放可复用的流程代理（Agent）定义文件，帮助团队按统一流程完成“从需求输入到标书产出”的全过程。

---

## 可用代理

### 1) 标书流程代理
- 文件：`agents/标书流程代理.md`
- 用途：读取 `标书/CLAUDE.md` 作为总控流程，按四步完成标书生产：
  1. 输入分析与路由
  2. 调用政务/工程/通用分析 skill
  3. 厂商筛选与评估
  4. 生成标书初稿

---

## 如何调用这个代理

在对话中直接表达你的目标，并附上输入材料。推荐指令示例：

- `请按 agents/标书流程代理.md 执行，从分析到标书初稿全流程完成。`
- `按标书流程代理执行，先做路由分析，再继续后续步骤。`
- `调用标书流程代理，输出项目分析、厂商推荐和标书初稿。`

---

## 输入示例

### A. 上传文件方式（推荐）
示例话术：
- `我已上传招标文件和需求附件，请按标书流程代理开始执行。`
- `基于我上传的招标PDF和评分细则，生成完整标书初稿。`

说明：
- 可上传 PDF、Word、图片、扫描件
- 图片/扫描件会先 OCR，再进入分析

### B. 本地路径方式
示例话术：
- `请分析这个文件并按代理流程执行：/mnt/user-data/uploads/某项目招标文件.pdf`
- `按代理流程处理以下路径：/mnt/user-data/uploads/项目A/`

说明：
- 支持单文件或目录路径
- 多文件会先汇总再分类路由

### C. 文本粘贴方式
示例话术：
- `以下是招标公告正文，请按代理流程生成标书：...`
- `我先贴评分办法和资格要求，请先做分析与路由。`

说明：
- 文本建议包含：项目概况、资格条件、评分办法、技术要求
- 信息不足时，代理会先提问补齐

---

## 建议输入模板（可复制）

```text
请按 agents/标书流程代理.md 执行全流程。
项目材料来源：<上传文件 / 本地路径 / 粘贴文本>
项目类型偏好：<政务服务类 / 工程实施类 / 不确定>
重点诉求：<中标率优先 / 价格优先 / 进度优先 / 质量优先>
是否需要厂商筛选：<是/否>
期望输出：<标书初稿 / 厂商推荐报告 / 全部>
```

---

## 输出物说明

代理完成后，通常会给出以下结果：
1. 路由分析结果（项目类型、判定依据、后续 skill）
2. 专业分析结果（评分点、结构拆解、模块建议、优化点）
3. 厂商推荐报告（排名、资质、参数、报价、风险、来源）
4. 标书初稿（分级标题、正式语言、可直接编辑）

## 常见问题

### 为什么没有直接生成标书？
通常是因为输入材料不完整（例如缺评分细则、资格门槛或预算信息）。代理会先补齐关键信息再继续。

### 可以跳过厂商筛选吗？
可以。在输入中写明“无需厂商筛选”，代理会直接进入标书生成阶段。

### 可以只做厂商推荐吗？
可以。直接说明“只执行厂商筛选与推荐”，代理会优先走该分支。
