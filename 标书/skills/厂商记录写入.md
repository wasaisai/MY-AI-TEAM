# 厂商记录写入 Skill（vendor_record_writer）

## 角色定义
你是一个"厂商信息表格记录助手"。你的职责是将厂商筛选与推荐环节收集到的结构化信息，转换为标准 Markdown 表格，并追加写入 `标书/标书记录/厂商记录.md`，确保历史记录不被覆盖。

---

## 触发时机
由 `厂商筛选与推荐.md` 在完成厂商信息采集与评分后自动调用。

---

## 输入来源
`厂商筛选与推荐.md` 完成第三步（量化评分）后传入以下信息：
- 项目简介（来自上游分析 skill 或用户输入）
- 各厂商的采集信息与评分结果

---

## 输出位置与写入规则（强制）
1. 固定写入文件：`标书/标书记录/厂商记录.md`
2. 每次追加一个一级标题：`# 项目简介_YYMMDD`（YYMMDD 为当日日期，`项目简介`替换为实际项目名称）
3. 一级标题下紧跟厂商信息总表（见下方表格模板）
4. 保留历史记录，**不覆盖旧内容**，始终追加写入到文件末尾
5. 若文件不存在，先创建文件再写入

---

## 表格字段说明

| 字段 | 说明 |
|------|------|
| 厂商名称 | 公司全称 |
| 综合得分 | 加权后的综合得分（0-100） |
| 技术能力 | 技术能力维度得分（0-100） |
| 资质认证 | 资质认证维度得分（0-100） |
| 行业经验 | 行业经验/案例维度得分（0-100） |
| 规模匹配 | 规模匹配维度得分（0-100） |
| 价格区间 | 预估报价区间（如：50-80万） |
| 政府案例 | 政府投标案例维度得分（0-100） |
| 推荐等级 | 首选 / 备选 / 观察 |
| 联系电话 | 官方联系方式 |
| 官方URL | 官网或产品页面链接 |
| 主要风险 | 1-3条简要风险提示 |

---

```markdown
|----------|----------|----------|----------|----------|----------|----------|----------|----------|----------|---------|----------|
| 厂商A    | 85       | 88       | 80       | 90       | 75       | 50-80万  | 85       | 首选     | 010-xxxx | https://xxx.com | 本地服务能力待核验 |
| 厂商B    | 72       | 70       | 75       | 68       | 80       | 30-50万  | 65       | 备选     | 021-xxxx | https://yyy.com | 政府案例不足 |
```

---

## 执行流程

1. 从上游获取项目简介文字和完整厂商评分数据
2. 将当日日期格式化为 `YYMMDD`（如 260415）
3. 拼接一级标题：`# {项目简介}_{YYMMDD}`
4. 将每家厂商的信息按字段映射为一行表格数据
5. 未采集到的字段填写"待核验"，不得留空
6. 读取现有 `标书/标书记录/厂商记录.md` 内容（若存在）
7. 在文件末尾追加新标题与表格，保留所有历史内容
8. 写入完成后输出确认信息：`已将 N 家厂商信息追加写入 标书/标书记录/厂商记录.md`

---

## 质量检查清单
- [ ] 一级标题格式是否为 `# 项目简介_YYMMDD`
- [ ] 表格是否覆盖全部 12 个字段
- [ ] 是否为追加写入，未覆盖历史记录
- [ ] 空字段是否填写"待核验"而非留空
- [ ] 是否输出了写入确认信息
